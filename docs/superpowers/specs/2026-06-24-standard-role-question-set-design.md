# 标准角色题库替换设计

## 目标

将当前角色配置直接替换为 5 个标准角色，并清理旧角色与旧 bot 数据，使 WebUI 中可直接 review 和微调新的标准题库。

标准角色：

- 产品经理
- 测试工程师
- 研发工程师
- 市场人员
- 运营人员

## 设计范围

本次替换范围包括：

- `roles`
- `role_documents`
- `role_questions`
- 旧 bot / channel / 初始化相关运行数据

保留：

- `playground.md`
- 系统代码与部署结构

不做兼容迁移，不保留旧默认角色集。

## 角色模型

每个标准角色包含两层：

1. 隐藏规则文档 `role.md`
2. 管理员初始化引导问题 `role_questions`

其中：

- `role.md` 用于沉淀该角色的专业默认规则与最佳实践
- `role_questions` 只承载需要管理员显式配置的个性化问题
- 不把专业最佳实践重复暴露为初始化问题

## 引导问题原则

- 各角色问题数允许差异化
- 每题只问一个问题
- 每题提供明确候选项
- 默认允许“其他，可直接描述”
- WebUI 中只暴露：
  - 问题标题
  - 选项文案
  - 是否默认添加其他

系统内部自动补：

- question key
- question type
- sort order
- enabled
- required
- option value

## 标准角色设计

### 1. 产品经理

隐藏规则聚焦：

- 默认先补齐背景、目标用户、核心问题
- 默认补齐范围、非范围、限制条件、风险点
- 涉及环信需求时默认检查 Console、IMM、计量计费、集群、开关灰度兼容性
- 输出偏评审与落地，不写空泛 PRD

引导问题聚焦：

- 交互方式
- 是否长期沉淀规则与文档
- 默认输出形态（PRD / 评审 / 用户故事等）
- 是否强调结构化结论
- 是否需要先给推荐方案
- 是否有额外工作规则

### 2. 测试工程师

隐藏规则聚焦：

- 优先从需求边界、异常流、兼容性和回归风险切入
- 默认补齐前置条件、测试范围、非测试范围、风险说明
- 输出测试用例时强调步骤、预期结果、边界与回归覆盖

引导问题聚焦：

- 交互方式
- 是否长期沉淀规则与测试资产
- 默认输出偏测试方案 / 测试用例 / 回归清单 / 缺陷分析
- 是否优先关注异常场景
- 是否强调兼容性与回归
- 是否有额外工作规则

### 3. 研发工程师

隐藏规则聚焦：

- 默认先澄清目标、输入输出、约束和依赖
- 优先给可实现方案，关注兼容性、回滚、灰度与边界
- 输出方案时强调模块边界、接口、数据流、异常处理

引导问题聚焦：

- 交互方式
- 是否长期沉淀规则与技术文档
- 默认输出偏技术方案 / 接口设计 / 任务拆解 / 排障分析
- 是否强调兼容性与回滚
- 是否默认给实现步骤
- 是否有额外工作规则

### 4. 市场人员

隐藏规则聚焦：

- 默认围绕用户价值、竞品差异、卖点表达、传播角度组织内容
- 强调结论清晰、信息可传播、避免技术化表达过重
- 输出分析时突出结论、支撑信息与建议动作

引导问题聚焦：

- 交互方式
- 是否长期沉淀规则与市场文档
- 默认输出偏竞品分析 / 卖点整理 / 活动文案 / 市场调研摘要
- 是否强调结论先行
- 是否偏外部表达口径
- 是否有额外工作规则

### 5. 运营人员

隐藏规则聚焦：

- 默认关注执行流程、资源协调、状态跟踪、异常处理与复盘
- 强调明确动作、责任、时间点和风险项
- 输出内容应服务于日常推进与跨团队协作

引导问题聚焦：

- 交互方式
- 是否长期沉淀规则与运营文档
- 默认输出偏执行方案 / SOP / 活动排期 / 复盘纪要
- 是否强调任务跟进与风险提醒
- 是否偏列表化和行动项
- 是否有额外工作规则

## 数据替换策略

执行时：

1. 清空旧 bot / channel / 初始化相关数据
2. 清空旧角色相关数据：
   - `roles`
   - `role_documents`
   - `role_questions`
3. 保留 `playground.md`
4. 写入 5 个标准角色
5. 写入各自 `role.md`
6. 写入各自 `role_questions`

## WebUI 预期结果

替换完成后：

- `角色管理` 只显示 5 个标准角色
- 每个角色都能打开详情
- 详情页可直接编辑：
  - 隐藏规则文档
  - 引导问题
- 管理员可在页面上微调后直接使用

## 验证

至少验证：

1. `GET /v1/roles` 只返回 5 个标准角色
2. 每个角色都有至少 1 个 `role.md`
3. 每个角色都有问题集
4. WebUI 角色列表正确展示 5 个角色
5. WebUI 角色详情可正常打开并显示对应文档与问题
6. 新建 bot 初始化时角色选择只显示这 5 个角色
